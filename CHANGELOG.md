# CHANGELOG

## [Unreleased]

### Добавлено
- **Режим планирования**: Добавлена кнопка ручного запуска планирования (📋) в интерфейсе чата
- **Улучшенная логика планирования**: Расширена поддержка ключевых слов для автоматического определения необходимости планирования
- **Заморозка FileWatcher**: Добавлены методы `freeze()`, `unfreeze()`, `isFrozen()` для предотвращения обработки изменений во время выполнения инструментов
- **Поддержка force_plan**: WebSocket сервер теперь поддерживает явный запрос планирования через параметр `force_plan`

### Изменено
- **frontend/chat.html**: Изменена структура формы чата, добавлена кнопка планирования
- **frontend/css/style.css**: Добавлены стили для кнопки планирования, изменена верстка формы
- **frontend/js/chat.js**: Реализована функция `sendMessage(forcePlan)` и обработчик кнопки планирования
- **src/FileWatcher.cpp**: Добавлена логика заморозки/разморозки
- **src/FileWatcher.h**: Добавлены объявления методов заморозки
- **src/WebSocketServer.cpp**: Улучшена логика определения необходимости планирования, добавлено логирование
- **src/tools/ToolManager.cpp**: Отключена блокировка мьютекса для предотвращения гонки с FileWatcher

### Технические детали
- Добавлен `std::atomic<bool> is_frozen` в FileWatcher для атомарной проверки состояния
- Улучшена обработка запросов WebSocket с поддержкой явного флага планирования
- Расширена поддержка ключевых слов для автоматического определения необходимости планирования (не только в начале строки)
