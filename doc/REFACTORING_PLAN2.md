# 🚀 План по рефакторингу проекта Smart Hammer Agent

---

## 🎯 Цели рефакторинга

1.  **Повышение качества кода:** Применение современных стандартов и паттернов проектирования.
2.  **Упрощение поддержки:** Улучшение читаемости, модульности и документирования кода.
3.  **Повышение надежности:** Внедрение автоматизированного тестирования и улучшенной обработки ошибок.
4.  **Оптимизация производительности:** Использование современных инструментов сборки и оптимизация клиентской части.

---

## 🗺️ Дорожная карта

Рефакторинг будет разделен на три основных направления: **Общие улучшения**, **Backend (C++)** и **Frontend (JavaScript)**.

### **Этап 1: Общие улучшения проекта (инфраструктура)**

| Задача | Описание | Зачем это нужно |
| :--- | :--- | :--- |
| **1.1. Настроить CI/CD** | Создать workflow для GitHub Actions, который будет собирать C++ проект и проверять JS-код линтером при каждом коммите. | Автоматизация проверок, гарантия работоспособности основной ветки. |
| **1.2. Улучшить документацию** | Обновить `README.md`: добавить секции по архитектуре, локальному запуску и сборке. | Упрощение входа для новых разработчиков и систематизация знаний о проекте. |
| **1.3. Внедрить Doxygen/JSDoc** | Настроить генерацию документации из комментариев в коде для C++ и JS. | Автоматическое создание актуальной документации по коду. |
| **1.4. Настроить vcpkg/Conan** | Перевести `FetchContent` на полноценный менеджер пакетов для C++. | Упрощение управления версиями зависимостей и сборки на разных платформах. |

---

### **Этап 2: Рефакторинг Backend (C++)**

| Задача | Описание | Зачем это нужно |
| :--- | :--- | :--- |
| **2.1. Внедрить юнит-тесты** | Добавить Google Test. Покрыть тестами базовую логику: `CodeParser`, `ToolManager`, отдельные инструменты. | Гарантия корректной работы бизнес-логики, предотвращение регрессий. |
| **2.2. Умные указатели** | Провести аудит кода на предмет использования "сырых" указателей (`new`/`delete`) и заменить их на `std::unique_ptr` и `std::shared_ptr`. | Автоматическое управление памятью, предотвращение утечек. |
| **2.3. Рефакторинг обработки ошибок** | Заменить возврат `nullptr` и кодов ошибок на механизм исключений или `std::optional`. | Создание более чистого и предсказуемого кода обработки ошибок. |
| **2.4. Применение SOLID** | Выделить интерфейсы (абстрактные классы) для `ContextIndexer`, `CodeParser`, `LLMProvider` и использовать инъекцию зависимостей. | Уменьшение связанности кода, упрощение тестирования и замены компонентов. |
| **2.5. Использование C++20** | Провести ревизию кода на предмет возможного использования новых возможностей: `std::ranges`, `std::span`, `std::format`. | Повышение читаемости и производительности кода. |
| **2.6. Анализ многопоточности** | Проверить `ThreadPool` и смежные классы на предмет потенциальных гонок данных и блокировок. | Повышение стабильности и надежности асинхронных операций. |

---

### **Этап 3: Рефакторинг Frontend (JavaScript)**

| Задача | Описание | Зачем это нужно |
| :--- | :--- | :--- |
| **3.1. Настроить сборщик Vite** | Создать `package.json`, установить Vite. Настроить сборку JS-модулей, минификацию и перенос статики. | Оптимизация фронтенда для production, быстрая разработка с HMR. |
| **3.2. Настроить ESLint и Prettier** | Добавить конфигурационные файлы `.eslintrc`, `.prettierrc` и скрипты для проверки и форматирования кода. | Поддержание единого стиля кода, автоматическое исправление ошибок. |
| **3.3. Очистить структуру JS** | Удалить устаревшие файлы (`chat.js`, `old_agent.js`, `socket.js` и т.д.). Перенести все новые модули в `frontend/js/agent/`. | Устранение путаницы и дублирования кода. |
| **3.4. Исправить импорты в HTML** | В `agent_v1.html` заменить `importScripts` на нативные `import` в `<script type="module">`. | Исправление текущей ошибки в консоли, переход на современный стандарт. |
| **3.5. Внедрить PostCSS** | Перенести Tailwind CSS с CDN в конфигурацию PostCSS, обрабатываемую через Vite. | Уменьшение размера CSS, удаление неиспользуемых стилей, ускорение загрузки. |
| **3.6. Компонентный подход (Lit)** | Внедрить библиотеку Lit. Разбить `agent_v1.html` на веб-компоненты (`chat-message`, `widget-plan`, `input-area`). | Улучшение переиспользуемости и управляемости UI, упрощение логики. |
| **3.7. Управление состоянием** | Создать простой класс-хранилище (`state-manager.js`) для централизованного управления состоянием UI (статус подключения, список сообщений). | Устранение прямой манипуляции DOM, создание единого источника правды. |

## 📊 Приоритеты

1.  **Высокий приоритет (Must Have):**
    *   Настройка Vite и ESLint/Prettier (3.1, 3.2).
    *   Исправление импортов в HTML (3.4).
    *   Внедрение юнит-тестов для C++ (2.1).
    *   Переход на умные указатели (2.2).

2.  **Средний приоритет (Should Have):**
    *   Настройка CI/CD (1.1).
    *   Рефакторинг обработки ошибок в C++ (2.3).
    *   Очистка структуры JS-файлов (3.3).
    *   Улучшение документации (1.2).

3.  **Низкий приоритет (Could Have):**
    *   Переход на компонентный подход с Lit (3.6).
    *   Внедрение `vcpkg` или `Conan` (1.4).
    *   Полное применение SOLID в C++ (2.4).

## 📝 Пример структуры модуля после рефакторинга (Frontend)

**`frontend/js/components/chat-message.js` (с использованием Lit)**

```javascript
import { LitElement, html, css } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('chat-message')
export class ChatMessage extends LitElement {
  @property({ type: String })
  role = 'user'; // 'user' или 'agent'

  @property({ type: String })
  text = '';

  static styles = css`
    .message {
      padding: 10px;
      margin: 5px;
      border-radius: 8px;
    }
    .user {
      background-color: #3b82f6;
      color: white;
      align-self: flex-end;
    }
    .agent {
      background-color: #1e293b;
      border: 1px solid #334155;
    }
  `;

  render() {
    return html`
      <div class="message ${this.role}">
        <p>${this.text}</p>
      </div>
    `;
  }
}
```

**Использование в `agent_v1.html`:**

```html
<chat-message role="user" text="Привет, Агент!"></chat-message>
<chat-message role="agent" text="Здравствуйте! Чем могу помочь?"></chat-message>
```
