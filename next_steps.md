# План развития «Smart Hammer»

## I. Улучшения пользовательского интерфейса (UX)

### 1. Drag-and-Drop для редактирования плана
**Задача:** Позволить пользователю не только редактировать текст шагов плана, но и менять их порядок простым перетаскиванием (drag-and-drop).
**Реализация:**
-   **Frontend:** Использовать легковесную JS-библиотеку (например, `SortableJS`) или нативный HTML Drag and Drop API для реализации перетаскивания `<li>` элементов в виджете `addPlanConfirmationWidget`.
-   **Backend:** Изменений не требуется, так как бэкенд получает уже готовый, отсортированный пользователем массив шагов.

### 2. Визуализация статуса `FileWatcher`
**Задача:** Добавить в интерфейс индикатор, который показывает, что `FileWatcher` активен и в данный момент переиндексирует файлы.
**Реализация:**
-   **Backend:** `FileWatcher` при начале и окончании переиндексации должен отправлять WebSocket-сообщения (`file_watcher_status: { status: 'indexing' | 'idle' }`).
-   **Frontend:** Поймать это сообщение и отобразить небольшой анимированный индикатор рядом со статусом соединения.

## II. Улучшения ядра агента

### 1. "Самолечение" агента с помощью Git
**Задача:** Научить агента автоматически отменять изменения в случае провала шага плана.
**Реализация:**
-   **Логика:** Если шаг, модифицирующий файлы (`edit_file`, `write_file`), завершается ошибкой, и пользователь выбирает "Отменить", агент должен выполнить команду `git checkout -- <имя_файла>`, чтобы вернуть файл к состоянию последнего коммита.
-   **Инструменты:** Потребуется новый внутренний инструмент `git_revert_file(path)` или расширение логики `handleErrorRecoveryConfirmation` в `WebSocketServer`.

### 2. Улучшенное логирование при ошибках
**Задача:** В случае ошибки в `processAgentLogic` или `AssistantRole` записывать в лог не только сообщение об ошибке, но и всю историю сообщений (`messages`), которая к ней привела.
**Реализация:**
-   **Backend:** В блоках `catch` и при обработке `response.step_failed` добавить в `SPDLOG_ERROR` дамп JSON-массива `session->history`. Это бесценно для отладки.

### 3. "Рабочий стол" агента (Agent's Scratchpad)
**Задача:** Дать агенту возможность сохранять промежуточные мысли, выводы и данные между шагами плана, чтобы принимать более взвешенные решения.
**Реализация:**
-   **Backend:** Добавить в `UserSession` новое поле `nlohmann::json scratchpad`.
-   **Инструменты:** Создать два новых инструмента: `write_to_scratchpad(key, value)` и `read_from_scratchpad(key)`.
-   **Промпт:** Обновить системный промпт, объяснив агенту, как использовать "рабочий стол" для сохранения важной информации (например, имен найденных функций, результатов анализа и т.д.).

## III. Долгосрочные цели
-   **Поддержка мультимодальности**: Возможность передавать агенту скриншоты ошибок.
-   **Оптимизация производительности индексации**: Использование многопоточности при генерации эмбеддингов.
-   **Веб-интерфейс для конфигурации**: Страница настроек для редактирования `config.json` прямо из браузера.
