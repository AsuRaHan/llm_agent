# Локальный AI-ассистент для анализа кода «Smart Hammer»

## Описание проекта


Это консольное приложение на C++, которое выступает в роли локального AI-помощника (агента), способного **понимать структуру вашего кода**. Оно индексирует кодовую базу, используя **Tree-sitter** для разбора кода на логические блоки (функции, классы), и создает для них "смысловые воспоминания" (эмбеддинги). Затем оно использует **высокопроизводительный поиск по графу (HNSW)** для нахождения наиболее релевантного контекста и отвечает на вопросы пользователя о проекте, используя большую языковую модель (LLM).

Проект спроектирован для работы с локально запущенным сервером LLM, совместимым с OpenAI API (например, `llama.cpp` в режиме сервера).

## Ключевые возможности

- **"Умная" индексация с Tree-sitter:** Вместо примитивного разбиения на куски по количеству символов, агент использует Tree-sitter для синтаксического анализа кода. Это позволяет выделять в качестве чанков целые функции, классы и другие логические блоки, что кардинально повышает качество контекста.
- **Высокопроизводительный поиск:** Для поиска по смыслу используется библиотека `Hnswlib`, реализующая алгоритм HNSW (Hierarchical Navigable Small World). Это обеспечивает практически мгновенный поиск по тысячам эмбеддингов даже на больших проектах.
- **Гибридное хранилище:** Индекс векторов хранится в компактном бинарном формате (`.bin`) для максимальной скорости загрузки, а метаданные (текст кода, пути к файлам) — в удобном JSON-формате.
- **Генерация ответов с контекстом (RAG):** Формирует промпт для LLM, включающий вопрос пользователя и найденные фрагменты кода, чтобы получить точный и основанный на контексте ответ.
- **Использование инструментов (Function Calling):** Агент может самостоятельно принимать решения о вызове C++ функций для получения дополнительной информации.
  - `read_file`: чтение файлов.
  - `list_directory`: просмотр содержимого директорий.
  - `code_search`: выполнение дополнительного семантического поиска.
  - `grep_search`: поиск по файлам с использованием регулярных выражений.
- **Конфигурация:** Все ключевые параметры (адрес сервера, таймауты, количество результатов) управляются через файл `config.json`.
- **Надежность:** Включает логику повторных запросов к серверу LLM при сбоях соединения.
- **Дружелюбный интерфейс:** Агент приветствует пользователя после сканирования проекта и готов к диалогу.

## Структура проекта

Проект состоит из нескольких ключевых компонентов:

- `main.cpp`: Точка входа, главный цикл приложения. Обрабатывает аргументы командной строки, инициализирует компоненты и управляет диалогом с пользователем.
- `ContextIndexer.cpp`: Отвечает за индексацию файлов, управление индексом HNSW, а также за поиск релевантных фрагментов (`findTopK`).
- `CodeParser.cpp`: Отвечает за синтаксический анализ кода с помощью Tree-sitter и разделение его на логические чанки.
- `EmbeddingClient.cpp`: Клиент для взаимодействия с эндпоинтом `/v1/embeddings` LLM-сервера. Отправляет фрагменты текста и получает их векторные представления.
- `AssistantRole.cpp`: Клиент для взаимодействия с эндпоинтом `/v1/chat/completions` LLM-сервера. Формирует промпты (как для ответов на вопросы, так и для генерации приветствия) и обрабатывает ответы модели.
- `src/tools/`: Директория, содержащая реализацию всех инструментов (`read_file`, `grep_search` и т.д.) и `ToolManager` для управления ими.
- `Config.cpp`: Загрузка и управление конфигурацией из `config.json`.
- `Logger.cpp`: Настройка логирования с помощью `spdlog` для записи событий в файл `app.log`.

## Зависимости

Проект использует следующие сторонние библиотеки:

- **nlohmann/json:** для удобной работы с JSON.
- **yhirose/cpp-httplib:** для выполнения HTTP-запросов к LLM-серверу.
- **gabime/spdlog:** для гибкого и быстрого логирования.
- **nmslib/hnswlib:** для сверхбыстрого приближенного поиска ближайших соседей (ANN).
- **tree-sitter/tree-sitter:** для построения синтаксических деревьев и структурного анализа кода.

Все зависимости управляются автоматически через `FetchContent` в `CMakeLists.txt`, поэтому их ручная установка не требуется.

## Сборка

Для сборки проекта необходим компилятор с поддержкой C++20 и установленный CMake (версия 3.15 или выше). Для Windows-пользователей в корне проекта есть скрипт `build.bat`, который автоматизирует эти два шага.

```bash
# 1. Сконфигурировать проект
cmake -S . -B build

# 2. Собрать проект
cmake --build build --config Release
```

Готовый исполняемый файл `Agent.exe` (или `Agent` в Linux/macOS) появится в корневой директории проекта.

## Запуск

### Требования

1.  **Запущенный LLM-сервер:** Убедитесь, что у вас запущен локальный сервер (например, `llama.cpp`), который предоставляет OpenAI-совместимый API по адресу, указанному в `config.json`. Сервер должен поддерживать эндпоинты:
    - `/v1/embeddings`
    - `/v1/chat/completions`
    - `/health` (для проверки соединения)

2.  **Файл конфигурации:** В корневой директории проекта должен находиться файл `config.json`.

### Пример `config.json`

```json
{
    "server": {
        "host": "localhost",
        "port": 8080,
        "api_key": "",
        "retry_count": 3,
        "retry_delay_ms": 500
    },
    "assistant": {
        "model_name": "any",
        "chat_completion_timeout_sec": 300,
        "max_tool_calls": 10
    },
    "embedding": {
        "model_name": "any",
        "max_text_length": 4096,
        "embedding_chunk_overlap": 512
    },
    "tools": {
        "enable_dangerous_tools": true
    },
    "logging": {
        "log_file_path": "agent.log",
        "log_to_console": true,
        "log_file_level": "trace",
        "log_console_level": "info"
    },
    "indexing": {
        "top_k_results": 5,
        "chunking_strategy": "tree-sitter-hybrid",
        "ignored_directories": [
            "build", ".git", ".vscode", "CMakeFiles", ".shdata"
        ],
        "ignored_extensions": [
            ".exe", ".obj", ".pdb", ".ilk", ".sln", ".vcxproj", ".filters", ".user",
            ".recipe", ".tlog", ".lastbuildstate", ".bin", ".stamp", ".cmake",
            ".json", ".log"
        ],
        "ignored_files": [
            ".gitignore"
        ]
    }
}
```

#### Описание параметров

*   **`server`**: Настройки подключения к LLM-серверу.
    *   `host`, `port`: Адрес и порт вашего локального сервера.
    *   `api_key`: (Опционально) Ваш API-ключ для удаленного сервиса. Если он указан, будет отправляться в заголовке Authorization: Bearer
    *   `retry_count`: Количество повторных попыток при сбое сетевого соединения.
    *   `retry_delay_ms`: Задержка между повторными попытками в миллисекундах.
*   **`embedding`**: Настройки для генерации эмбеддингов.
    *   `model_name`: Имя модели для эмбеддингов. Для `llama.cpp` можно оставить `"any"`.
    *   `max_text_length`: Максимальная длина текста (в символах) для одного чанка при разбиении.
    *   `embedding_chunk_overlap`: Нахлест для чанков, которые создаются из слишком больших семантических блоков кода.
*   **`assistant`**: Настройки для чат-комплишенов (генерации ответов).
    *   `model_name`: Имя модели для генерации ответов. Для `llama.cpp` можно оставить `"any"`.
    *   `chat_completion_timeout_sec`: Таймаут ожидания ответа от модели в секундах.
    *   `max_tool_calls`: Максимальное количество вызовов инструментов в рамках одного запроса пользователя.
*   **`indexing`**: Настройки процесса индексации.
    *   `top_k_results`: Количество наиболее релевантных чанков, которые будут переданы в LLM для генерации ответа.
    *   `chunking_strategy`: Стратегия разбиения файлов на чанки.
        *   `"fixed"`: Простое разбиение по `chunk_size`.
        *   `"tree-sitter"`: "Умное" разбиение на функции и классы с помощью Tree-sitter.
        *   `"tree-sitter-hybrid"`: (Рекомендуется) Разбиение с помощью Tree-sitter с последующей генерацией краткого саммари для каждого чанка с помощью LLM. Это улучшает релевантность поиска.
    *   `ignored_directories`, `ignored_extensions`, `ignored_files`: Списки каталогов, расширений и файлов, которые следует игнорировать при индексации.

### Команда запуска

Вы можете запустить агент, указав путь к директории проекта, который нужно проанализировать:

```bash
# Анализировать текущую директорию
./Agent

# Анализировать конкретную директорию
./Agent /path/to/your/project
```

После запуска агент проиндексирует файлы и будет готов отвечать на ваши вопросы о коде.

### Команда запуска llama.cpp для работы агента

пример bat фала для запуска llama.cpp на локальном сервере.
параметры `--embedding --pooling mean` должны быть обязательными. без них этот ИИ агент работать не будет.

```bat
@echo off
cd /d "E:\ai_training\llama.cpp\build\bin\Release"

set SERVER_BIN=llama-server.exe
set MODEL_PATH=E:\ai_training\models\Tesslate\OmniCoder-9B-GGUF\omnicoder-9b-q8_0.gguf

:: Оптимизировано под 16GB VRAM
"%SERVER_BIN%" ^
    --model "%MODEL_PATH%" ^
    --n-gpu-layers 99 ^
    --ctx-size 232768 ^
    --flash-attn on ^
    --cache-type-k q4_0 ^
    --cache-type-v q4_0 ^
    --batch-size 2048 ^
    --mlock --embedding --pooling mean


echo Сервер на запущен. Нажмите Ctrl+C для остановки.
pause
```
