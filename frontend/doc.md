# Руководство пользователя «Smart Hammer»

## 1. Введение

«Smart Hammer» — это ваш персональный AI-ассистент для работы с кодом. Он не просто ищет по ключевым словам, а **понимает семантику** вашего проекта. Агент индексирует кодовую базу, разбирая ее на логические блоки (функции, классы), и использует эти знания, чтобы давать точные ответы, выполнять сложные задачи и даже самостоятельно писать и исправлять код.

## 2. Ключевые возможности

- **Семантическое понимание кода**: Благодаря `Tree-sitter`, агент видит код так же, как и вы — в виде функций, классов и модулей, а не простого текста.
- **Мгновенный поиск по смыслу**: Находит наиболее релевантные участки кода для ответа на ваш вопрос, используя векторный поиск.
- **Автономный режим "План-и-Выполнение"**: Для сложных задач (например, "реализуй новую фичу" или "исправь баг") агент сначала составляет пошаговый план, который вы можете утвердить и отредактировать, а затем выполняет его автономно.
- **Интерактивное восстановление после ошибок**: Если на каком-то шаге возникает ошибка, агент не сдается, а предлагает вам варианты решения: повторить, пропустить шаг или перепланировать задачу.
- **Широкий набор инструментов**: Агент умеет не только читать код, но и:
    - Искать информацию в интернете (`web_search`).
    - Читать файлы по URL (`read_url`).
    - Исследовать структуру проекта (`list_directory`).
    - Выполнять поиск по регулярным выражениям (`grep_search`).
    - Редактировать и создавать файлы (`edit_file`, `write_file`).
    - Применять патчи (`apply_diff`).
    - Выполнять команды в терминале (`execute_shell_command`).
- **Безопасность**: "Опасные" инструменты, которые могут изменить ваш проект, всегда требуют вашего подтверждения перед выполнением.
- **Автоматическое обновление индекса**: Агент следит за изменениями в файлах и автоматически переиндексирует их в фоновом режиме.

## 3. Как пользоваться

### 3.1. Запуск

1.  **Запустите LLM-сервер**: Убедитесь, что у вас запущен локальный сервер (например, `llama.cpp`) с поддержкой эмбеддингов (`--embedding`). Пример запуска есть в `README.md`.
2.  **Запустите Агента**: Откройте терминал в папке с проектом и выполните:
    ```bash
    # Для анализа проекта в текущей папке
    ./Agent.exe .
    # Или укажите путь к другому проекту
    ./Agent.exe /path/to/your/project
    ```
3.  **Откройте веб-интерфейс**: Перейдите в браузере по адресу, указанному в консоли (обычно `http://localhost:9000`).

### 3.2. Взаимодействие с агентом

В интерфейсе чата вы можете общаться с агентом.

#### Простые вопросы

Вы можете задавать вопросы о коде, просить объяснить работу функции или найти, где используется определенный класс.

> **Пример:** "Где в проекте используется класс WebSocketServer?"
> **Пример:** "Объясни, как работает метод `processAgentLogic`."

#### Сложные задачи (Режим планирования)

Чтобы активировать автономный режим, начните ваш запрос с одного из ключевых слов-действий:

- `исправь`
- `реализуй`
- `перепиши`
- `добавь`
- `удали`
- `рефактор`
- `оптимизируй`
- `создай`

> **Пример:** "Добавь в класс `FileWatcher` новый метод для игнорирования временных файлов."
> **Пример:** "Исправь ошибку с доступом к файлу в `ToolManager`."

В ответ на такой запрос агент сначала предоставит вам **редактируемый план действий**. Вы сможете изменить, удалить или добавить шаги, прежде чем нажать "Утвердить и запустить". После этого агент начнет автономное выполнение, отчитываясь о прогрессе.

## 4. Как это работает (под капотом)

1.  **Индексация**: При первом запуске `ContextIndexer` сканирует ваш проект. С помощью `CodeParser` и `Tree-sitter` он разбивает каждый файл на осмысленные части (функции, классы).
2.  **Создание "Воспоминаний"**: Каждый такой "чанк" кода отправляется через `EmbeddingClient` на LLM-сервер, который превращает его в вектор (эмбеддинг) — математическое представление смысла этого кода.
3.  **Хранение**: Эмбеддинги сохраняются в быстром векторном индексе `hnswlib` (`.shdata/index.bin`), а метаданные (где какой чанк находится) — в `.shdata/index_meta.json`.
4.  **Поиск (RAG)**: Когда вы задаете вопрос, он тоже превращается в вектор. `ContextIndexer` мгновенно находит в индексе наиболее "похожие" по смыслу векторы кода и передает их в качестве контекста для LLM.
5.  **Мозг Агента (`AssistantRole`)**: Получив ваш вопрос и релевантный код, агент решает, что делать:
    - Если информации достаточно, он генерирует ответ.
    - Если информации мало, он использует один из своих инструментов (например, `read_file`, чтобы прочитать файл целиком).
    - Если задача сложная, он генерирует план и ждет вашего одобрения.

Эта комбинация семантического понимания кода, быстрого поиска и способности к автономным действиям и делает «Smart Hammer» мощным помощником в разработке.
