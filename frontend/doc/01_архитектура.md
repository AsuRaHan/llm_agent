# 🏗️ Архитектура Frontend LLM Agent

## 📋 Обзор

Этот документ описывает архитектуру нового фронтенда для LLM Agent, включая структуру проекта, основные классы, потоки данных и взаимодействия между компонентами.

---

## 🎯 Цели архитектуры

- Создать масштабируемую и поддерживаемую архитектуру
- Обеспечить четкое разделение ответственности между модулями
- Поддержать тестирование и отладку
- Обеспечить возможность расширения функционала

## 🏗️ Основные классы

### 1. App (Основное приложение)

```
App
├── Конструктор
│   ├── socketManager: SocketManager
│   ├── sessionManager: SessionManager
│   ├── messageRenderer: MessageRenderer
│   ├── widgetFactory: WidgetFactory
│   └── errorHandler: ErrorHandler
│
├── Методы
│   ├── init()              # Инициализация приложения
│   ├── startChat()         # Запуск чата
│   ├── handleUserMessage() # Обработка сообщения пользователя
│   ├── handleAgentMessage() # Обработка ответа агента
│   ├── render()            # Рендеринг UI
│   └── destroy()           # Очистка ресурсов
│
└── События
    ├── onMessageReceived
    ├── onWidgetCreated
    └── onError
```

**Ответственность:** Координация всех компонентов приложения, управление жизненным циклом.

---

### 2. SocketManager (WebSocket менеджер)

```
SocketManager
├── Конструктор
│   ├── sessionId: string
│   ├── ws: WebSocket
│   ├── url: string
│   ├── reconnectAttempts: number
│   ├── maxReconnectAttempts: number
│   ├── reconnectDelay: number
│   ├── messageHandlers: Array<Function>
│   ├── errorHandlers: Array<Function>
│   ├── isConnecting: boolean
│   └── pendingMessages: Array
│
├── Методы
│   ├── connect()              # Подключение
│   ├── send(type, data)       # Отправка
│   ├── onMessage(handler)     # Подписка на сообщения
│   ├── onError(handler)       # Подписка на ошибки
│   ├── handleClose()          # Обработка закрытия
│   ├── handleError(error)     # Обработка ошибок
│   └── handleMessage(message) # Обработка входящих сообщений
│
└── События
    ├── onopen
    ├── onclose
    ├── onmessage
    └── onerror
```

**Ответственность:** Управление WebSocket подключением, отправка/получение сообщений, реконнект.

---

### 3. SessionManager (Менеджер сессий)

```
SessionManager
├── Конструктор
│   ├── storage: Storage
│   ├── prefix: string
│   └── sessionId: string
│
├── Методы
│   ├── getSessionId()      # Получение ID сессии
│   ├── createSession()     # Создание новой сессии
│   ├── saveState()         # Сохранение состояния
│   ├── loadState()         # Загрузка состояния
│   ├── clearSession()      # Очистка сессии
│   ├── getState()          # Получение состояния
│   └── setState()          # Установка состояния
│
└── События
    ├── onSessionCreated
    └── onSessionExpired
```

**Ответственность:** Управление сессиями пользователя, сохранение/загрузка состояния.

---

### 4. MessageRenderer (Рендеринг сообщений)

```
MessageRenderer
├── Конструктор
│   ├── container: HTMLElement
│   ├── autoScroll: boolean
│   └── maxMessages: number
│
├── Методы
│   ├── renderWelcome()     # Рендер приветствия
│   ├── renderUserMessage() # Рендер сообщения пользователя
│   ├── renderAgentMessage() # Рендер сообщения агента
│   ├── renderSystemMessage() # Рендер системного сообщения
│   ├── renderError()       # Рендер ошибки
│   ├── renderMarkdown()    # Рендер markdown
│   ├── clearMessages()     # Очистка сообщений
│   ├── scrollToBottom()    # Авто-скролл
│   └── getMessages()       # Получение сообщений
│
└── Свойства
    ├── messages: Array
    └── scrollContainer: HTMLElement
```

**Ответственность:** Рендеринг сообщений в чате, форматирование, авто-скролл.

---

### 5. WidgetFactory (Фабрика виджетов)

```
WidgetFactory
├── Конструктор
│   ├── container: HTMLElement

│   └── registry: Map
│
├── Методы
│   ├── renderPlanWidget()  # Виджет плана
│   ├── renderProgressWidget() # Виджет прогресса
│   ├── renderConfirmationWidget() # Виджет подтверждения
│   ├── renderErrorWidget() # Виджет ошибки
│   ├── createWidget()      # Создание виджета
│   ├── removeWidget(id)    # Удаление виджета
│   ├── updateWidget(id, data) # Обновление виджета
│   └── getWidget(id)       # Получение виджета
```

**Ответственность:** Создание и управление виджетами, регистрация типов виджетов.

### 6. ErrorHandler (Обработчик ошибок)

```
ErrorHandler
```

**Ответственность:** Глобальная обработка ошибок, уведомления, механизмы восстановления.

## 🔄 Потоки данных

### 1. Поток отправки сообщения

```
1. Пользователь вводит сообщение
   │
   ▼
2. App.handleUserMessage(message)
   │
   ▼
3. SessionManager.saveState(message)
   │
   ▼
4. SocketManager.send('message', { text: message })
   │
   ▼
5. Ожидание ответа от сервера
   │
   ▼
6. Получение ответа
   │
   ▼
7. SocketManager.handleMessage(response)
   │
   ▼
8. App.handleAgentMessage(response)
   │
   ▼
9. MessageRenderer.renderAgentMessage(response)
   │
   ▼
10. SessionManager.saveState(response)
   │
   ▼
11. UI обновлен
```

### 2. Поток создания виджета

```
1. Получение данных для виджета
   │
   ▼
2. WidgetFactory.createWidget(type, data)
   │
   ▼
3. Создание экземпляра виджета
   │
   ▼
4. Рендеринг HTML
   │
   ▼
5. Привязка событий
   │
   ▼
6. Добавление в контейнер
   │
   ▼
7. Добавление в Map
   │
   ▼
8. UI обновлен
```

### 3. Поток обработки ошибок

```
1. Возникновение ошибки
   │
   ▼
2. ErrorHandler.handleError(error, event)
   │
   ▼
3. Логирование ошибки
   │
   ▼
4. Вызов обработчиков
   │
   ▼
5. Отображение уведомления
   │
   ▼
6. Определение типа ошибки
   │
   ▼
7. Получение recovery options
   │
   ▼
8. Отображение модального окна (при необходимости)
   │
   ▼
9. Пользователь выбирает действие
   │
   ▼
10. Выполнение recovery action
```

## 📊 Состояния системы

### Состояния WebSocket

```
CLOSED (3) → CONNECTING (0) → OPEN (1) → CLOSING (2) → CLOSED (3)
```

### Состояния сессии

```
┌─────────────────┐
│  Session Start  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Create Session │
│  Generate ID    │
│  Save Initial   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Active State   │
│  Save Updates   │
│  Check Expiry   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Session End    │
│  Clear Data     │
│  New Session    │
└─────────────────┘
```

### Состояния виджетов

```
Created → Rendered → Active → Updated → Removed
```

## 🔐 Безопасность

### 1. Валидация данных

- Все входящие данные проверяются на корректность
- Проверка типов данных
- Проверка длины строк
- Проверка на XSS-атаки

### 2. Хранение данных

- Использование `localStorage` для сессий
- Шифрование чувствительных данных (опционально)
- Ограничение размера сессий

### 3. WebSocket безопасность

- Использование `wss://` для production
- Проверка CORS заголовков
- Валидация WebSocket URL

### 4. Обработка ошибок

- Глобальные обработчики ошибок
- Логирование без утечки данных
- Recovery options для пользователя

## 🚀 Оптимизация

### 1. Кэширование

- Кэширование часто используемых данных
- Кэширование отформатированного контента

### 2. Ленивая загрузка

- Загрузка компонентов по требованию
- Ленивая загрузка виджетов

### 3. Debounce/Throttle

- Debounce для отправки сообщений
- Throttle для частых событий

### 4. Виртуальный скроллинг

- Для больших списков сообщений

## 📝 Best Practices

### 1. Модульность

- Каждый класс отвечает за одну задачу
- Минимальная зависимость между модулями
- Четкие интерфейсы

### 2. Тестируемость

- Изолированные компоненты
- Mock-объекты для зависимостей
- Покрытие тестами

### 3. Производительность

- Минимизация перерисовок
- Использование CSS переменных
- Оптимизация рендеринга

### 4. Поддержка

- Четкая документация
- Консистентный код
- Семантические имена
